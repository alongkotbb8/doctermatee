## 2. รีวิวลูกค้าในหน้าสินค้า (UGC) — `/products/[slug]`

| เทส | ผลที่ควรได้ |
|-----|------------|
| เลื่อนลงท้ายหน้าสินค้า (ยังไม่มีรีวิว) | ขึ้น "ยังไม่มีรีวิว เป็นคนแรก" + ปุ่ม "เขียนรีวิว" |
| กด "เขียนรีวิว" ขณะ**ไม่ login** | ขึ้น "เข้าสู่ระบบเพื่อเขียนรีวิว" + ปุ่มไป login |
| login แล้วเขียนรีวิว (ดาว + ข้อความ) | ส่งได้ → ขึ้น "รออนุมัติ" |
| ส่งรีวิวสินค้า**เดิมซ้ำ** | ขึ้น "คุณได้รีวิวสินค้านี้ไปแล้ว" (กันซ้ำ) |
| `/admin/product-reviews` | เห็นรีวิว pending → กดอนุมัติ → โผล่หน้าสินค้า |
| รีวิวจากคนที่**เคยซื้อผ่านเว็บ** | มี badge "✓ ผู้ซื้อจริง" |
| รีวิวจากคนที่**ไม่เคยซื้อผ่านเว็บ** | แสดงปกติ แต่ **ไม่มี** badge (ตั้งใจ — ใครก็รีวิวได้) |
| หลังมีรีวิวอนุมัติแล้ว | หน้าสินค้าโชว์คะแนนเฉลี่ย + แท่งกระจายดาว + การ์ดรีวิว |

## 3. กระทู้ถามตอบ (QAPage) — `/qa`

| เทส | ผลที่ควรได้ |
|-----|------------|
| เปิด `/qa` | เห็น 2 กระทู้ตัวอย่าง + ปุ่ม "ตั้งคำถามใหม่" |
| เปิดกระทู้ | เห็นคำถาม + คำตอบ + badge "คำตอบทางการ" (ทีมเภสัชกร) |
| ตั้งคำถาม / ตอบ ขณะไม่ login | ขึ้นให้เข้าสู่ระบบ |

| `/admin/qa` | แท็บคำถาม/คำตอบ → อนุมัติ/ปฏิเสธ/ลบ + **ตั้งคำตอบทางการ** |
| คำตอบที่ตั้งเป็น "ทางการ" | ขึ้น badge + เป็น acceptedAnswer ใน schema |

| จุด | เทส | ผลที่ควรได้ |

## 6. SEO / Schema (ทดสอบด้วย Google Rich Results Test)

| URL | ควรเจอ |
|-----|--------|
| `/reviews/vitamin-c-yi-ho-nai-di-2026` | Product + Review (ดาว) + Article ✅ ผ่านแล้ว |
| `/qa/vitamin-c-kin-kap-collagen-dai-mai` | QAPage |
| `/products/[slug]` | Product (+ ดาว เมื่อมีรีวิวอนุมัติแล้ว) |
| `/robots.txt` | อนุญาต GPTBot, ClaudeBot, anthropic-ai, Google-Extended, PerplexityBot, CCBot |
| `/sitemap.xml` | มี /reviews, /qa, สินค้า, บทความ ครบ |

## 7. พฤติกรรมที่ "ตั้งใจ" — อย่ารายงานเป็นบั๊ก

- **หน้าแรก `/` ไม่มี rich result** — ปกติ (schema อยู่หน้ารายละเอียด ไม่ใช่หน้ารวม)
- **FAQPage ไม่ขึ้นเป็น rich result แยกใน Google** — Google จำกัด FAQ ให้เฉพาะเว็บราชการ/สุขภาพทางการ (markup ยังถูกต้อง อ่านได้โดย AI/Bing)
- **หน้าสินค้ายังไม่มีดาว** — เพราะยังไม่มีรีวิวลูกค้าที่อนุมัติ (ห้ามใส่ดาวปลอมตาม policy Google)
- **เนื้อหา pending ไม่แสดงหน้าร้าน** — ต้องอนุมัติก่อน (by design)
- **ต้อง login ก่อนรีวิว/ถาม/ตอบ** — กันสแปม (ไม่ต้องซื้อก่อนก็รีวิวได้)
